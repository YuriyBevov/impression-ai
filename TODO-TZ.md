## 1. 📚 База знаний

Интерфейс работы с Qdrant.

### Главная страница Базы знаний:
- Поиск по коллекциям
- Поиск по точкам (записям) во всех коллекциях
- Возможность зайти в нужную коллекцию

### Внутри коллекции:
- **Просмотр записей** (список всех точек коллекции)
- **Поиск по ключу** — найти запись по определённому полю
- **Создание записи вручную** — ввод данных через форму
- **Редактирование** существующей записи
- **Удаление** записи
- **Дополнение** записи

### Загрузка документов для авто-заполнения:
- Загрузка текстового файла или документа
- Выбор модели ИИ для анализа
- Модель составляет пары (записи) на основе документа/текста
- **Предпросмотр перед сохранением:** возможность отредактировать сгенерированные записи, удалить ненужные
- Подтверждение → сохранение в выбранную коллекцию

### Автоматическая система дедупликации:
- Поиск дубликатов в базе данных
- Чистка дубликатов
- Предотвращение накопления дублирующихся записей

## 2. 🔄 Сделать перевод

Интерфейс отправки текстов в n8n для перевода через воронку Impression.

### Хедер страницы:
- Выбор клиента, для которого делается перевод (список из раздела Клиенты)
- При выборе клиента — приоритет на его базу знаний (см. архитектуру)

### Поле для ввода текста:
- Вставка скопированного текста в текстовое поле
- Отправка в n8n → воронка переводчика → получение результата

### Загрузка документа (на будущее):
- Загрузка файла для отправки в n8n
- Документ проходит ту же воронку перевода
- Поддерживаемые форматы:
  - Текстовые форматы
  - PDF
  - Изображения

## 🏗️ Архитектура базы знаний

### Иерархия коллекций (приоритет):
1. **Коллекция клиента** — наивысший приоритет. У каждого клиента своя коллекция с терминами и стилистикой.
2. **Общая коллекция (glossary)** — если в коллекции клиента нет термина, используется общая.
3. **Остальные коллекции** — ниже по приоритету.

### Влияние на перевод:
- При переводе от имени клиента → воронка n8n сначала ищет в коллекции клиента
- Если термин найден там — используется он (переопределяет общий glossary)
- Если нет — падает в общий glossary

### Что это значит для UI:
- При создании клиента, возможно, нужно автоматически создавать его коллекцию
- При редактировании Базы знаний — видеть, какая коллекция чья (общая или клиентская)
- В интерфейсе перевода — выбранный клиент определяет, какая коллекция будет приоритетной
